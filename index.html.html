Design System Rules

Everything you need to know before building a WOL page.

🏗 How Pages Are Built

Single HTML File

Every WOL page is one self-contained HTML file. All CSS inside a <style> tag. No external stylesheets, no separate JS files.

The Shell Template

Every page starts from _SHELL_COMPLETE.html. Contains site header, footer, mobile nav, back-to-top, and all base CSS variables. Never modify the shell.

Build Workflow

1. Read rules → 2. Find blueprint → 3. Copy shell → 4. Read components → 5. Build → 6. Verify shell unchanged → 7. Output.

🎨 Colors

Never hardcode hex values outside :root. Every color must reference a --wol-* variable.

TokenHexWhat It's For
--wol-heritage#003B44Header/footer backgrounds, headings, dark sections
--wol-green#00D875Button fills, active tab underlines on dark backgrounds
--wol-paper#F2F2EBBackground surfaces (never use as text)
--wol-link#336CC7Links and article type labels on light backgrounds
--wol-lilac#4885F7Charts and icon backgrounds (not for text)
--wol-neutral-900#000000Body text on light backgrounds
--wol-neutral-700#5D5E5CSecondary text: dates, meta, authors
--wol-neutral-200#E5E4E0Borders and dividers
--wol-white#FFFFFFText on dark backgrounds, page background
--wol-negative#CF1322Errors, closed-access badges
--wol-positive-text#256A09Open-access badge text, success labels
--wol-cyan#00EDE1Charts, secondary accent. Not for text on white.
--wol-lime#98F72DCharts, tertiary accent. Not for text on white.
--wol-sage-400#A3BFB5Alternate section backgrounds
--wol-neutral-600#8C8B89Large icons/decorative only. Not for text.
--wol-neutral-500#ADACA8Disabled states, decorative icons only. Never for readable text.

Green budget: Only 2–3 green elements per viewport. Green is for interactive things only. Never as text on white (1.9:1 contrast).

🔤 Typography

Inter for everything. IBM Plex Mono only for labels/badges/buttons — always uppercase.

TokenSizeUse For
--fs-body16pxAll readable content
--fs-secondary15pxSupporting text, summaries
--fs-ui14pxNav, tabs, buttons, inputs only
--fs-meta13pxDates, DOIs, affiliations
--fs-label12pxSmall labels
--fs-label-sm11pxPlex Mono labels
--fs-label-xs10pxMicro labels

Never use --fs-ui on readable content. Headings 28px+ are weight 300 (light).

📏 Spacing

Only font-size uses rem. Everything else in px. border-radius: 4px cards, 2px badges. Never 8px+. Use var(--wol-elevation-1) for card shadows.

⚙ JavaScript

Vanilla JS only, inline, under 40 lines. Allowed: tab switching, accordion, modal, dropdown, tooltips, scroll toggles. No API calls, no localStorage.

🔒 Shell Rules (Zero Tolerance)

Header, footer, mobile nav, back-to-top, CSS variables: character-for-character from shell. Logo URLs contain commit hash d86649612e3e. Nav order: Journals, Books, Browse, Resources.

✅ Pre-Delivery Checklist

☐ Shell verbatim ☐ Logo commit hash ☐ Nav order ☐ Zero hardcoded hex ☐ All font-sizes use vars ☐ No --fs-ui on content ☐ html 16px locked ☐ Green budget ☐ Green never text on white ☐ --wol-neutral-700 for secondary text ☐ border-radius 4px cards ☐ elevation-1 cards ☐ Padding on inner elements ☐ overflow-x hidden ☐ Plex Mono uppercase ☐ Blueprint followed

## Content Guidelines

- Every card needs: Volume, Issue number, Date (month + year)
- Optional: Special issue label, article count, download count
- Hover state: gradient bar animates in from left

- Show 2–4 issues per year (quarterly or monthly journals vary)
- For prototypes: show 3 most recent years with 4 issues each

- Count format: "Showing **48** issues"

- View toggle is optional (list view not typically used for issue archives)

Sticky TOC + Authors

The article page sidebar must use <aside class="article-sidebar"> (not .sidebar). This uses display: block instead of display: flex, which allows the CSS grid to stretch the aside to match the main column height, giving position: sticky room to scroll.

✓ Correct
<aside class="article-sidebar">
  <nav class="article-toc">...</nav>
  <div class="sidebar-card">...</div>
</aside>
✗ Wrong
<aside class="sidebar">
  <div class="article-toc-wrapper">
    <nav class="article-toc">...</nav>
  </div>
</aside>

Why: .sidebar uses display: flex which collapses height. .article-sidebar uses display: block so the grid stretches it to full main-column height, giving sticky proper scroll distance.
